# Руководство для Codex

## Контекст и архитектура

- Перед изменением кода прочитайте `ARCHITECTURE.md`: минимум разделы **Source Of Truth**, **Critical Invariants** и раздел, относящийся к задаче.
- `ARCHITECTURE.md` — карта проекта, а исходный код — источник истины при расхождении. Сообщите пользователю о найденном расхождении.
- Не обновляйте `ARCHITECTURE.md` автоматически. Меняйте его только по прямой просьбе пользователя или при явной просьбе зафиксировать новое архитектурное состояние.
- Для краткого запуска UI/API используйте `README.md`; детальные операционные инструкции находятся в `Docs/*_runbook.md`.

## Неподвижные правила проекта

- Bitrix-интеграция и API-адаптер работают только на чтение CRM. Не добавляйте операции записи в Bitrix.
- Не коммитьте `.env`, ключи, webhook URL, персональные данные или содержимое `reports/`. Считайте `reports/` чувствительными локальными данными.
- Все текстовые файлы — UTF-8. Кириллица намеренна: не заменяйте её транслитом или Unicode-escape. При записи JSON используйте `ensure_ascii=False`.
- Не обходите change detection прямым запуском LLM-анализаторов без явно запрошенного пользователем `--allow-direct-llm`.
- Не создавайте Markdown-отчёт, пока LLM JSON не прошёл валидацию. Не выдавайте правила OKF за факты конкретного лида или сделки.
- Пайплайны `leads` и `deals` похожи, но их JSON-контракты, prompts и renderer-правила различаются — проверяйте оба контура при общих изменениях.

## Рабочий процесс

- Сначала найдите source-of-truth файл для нужного контура через `ARCHITECTURE.md`; вносите минимальное изменение в существующую структуру.
- Не запускайте рабочие пайплайны, обращения к Bitrix или OpenAI без необходимости задачи: они используют внешние сервисы, могут занять время и создать локальные артефакты.
- Изменение UI обычно требует согласованной проверки `frontend/src/App.tsx`, `frontend/src/api.ts`, `api/app.py` и `api/jobs.py` — но меняйте только затронутые файлы.
- Обновляйте `README.md`, если меняется короткий способ запуска; runbook — если меняется соответствующая ручная процедура.

## Проверка изменений

- После изменения Python-логики запустите: `.\venv\Scripts\python.exe -m unittest discover -s tests`.
- После изменения frontend запустите в `frontend/`: `npm run lint` и `npm run build`.
- Если команда не запускалась, явно укажите причину в результате.
